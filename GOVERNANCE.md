# Governance

## Project Status

WhatTime is an actively maintained open source project. It is currently maintained by its original author.

## Decision Making

WhatTime uses a **Benevolent Dictator For Life (BDFL)** model. The project founder (@jeromecoloma) makes final decisions on direction, feature acceptance, and releases.

For day-to-day contributions:

- Bug fixes and documentation improvements are welcome via pull request and are merged at maintainer discretion
- New features should be discussed in a GitHub issue before work begins — this avoids wasted effort on changes that won't be accepted
- Breaking changes require explicit maintainer sign-off

## Maintainers

See [MAINTAINERS.md](MAINTAINERS.md) for the current list of active maintainers and their areas of responsibility.

## Adding Maintainers

New maintainers may be invited by the current maintainers after demonstrating sustained, high-quality contributions over time. There is no formal process — it is at maintainer discretion.

## Removing Maintainers

A maintainer who is inactive for more than 6 months may be moved to an "emeritus" status. Maintainers may also step down voluntarily by notifying the team.

## Project Abandonment

If the project becomes unmaintained:

1. The README will be updated to reflect that status
2. Maintainer access will not be granted to unknown parties
3. The repository will remain public and forkable

If you are interested in taking over stewardship of an unmaintained WhatTime, open an issue and make the case.

## Scope

WhatTime is intentionally minimal: a menubar-style timezone converter with natural language input. Contributions that expand the scope significantly (e.g., syncing with external services, adding accounts/login, building a companion iOS app) are out of scope and will not be accepted regardless of implementation quality.
